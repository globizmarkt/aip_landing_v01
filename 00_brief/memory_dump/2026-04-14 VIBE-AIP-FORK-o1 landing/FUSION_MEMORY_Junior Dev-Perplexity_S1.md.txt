## 4. DECISIONES DESCARTADAS Y POR QUÉ

- Texto hardcodeado en HTML/JS — viola R4 y rompe `window.LuxI18n`. [file:1][file:2]
- Colores hex inline — viola R3 y descentraliza tokens visuales. [file:1][file:2]
- Shadow DOM / `attachShadow()` — rompe Light DOM y la cascada global. [file:1][file:2]
- “Login como Paso 1” — contradice Gestación Local / Adopción Tardía. [file:1][file:2]
- Microcopy técnico tipo “Error 500” — no pasa el filtro de Alta Banca Suiza. [file:1]
- Suposiciones de rutas sin tree.txt — bloqueadas por el protocolo de la planta. [file:1]
- Overwriting de namespaces locales con bloques JSON separados — generaba error de sintaxis por múltiples raíces; se corrigió fusionando correctamente dentro del mismo objeto. [file:2]

## 8. OBJETOS GLOBALES Y FUENTE DE VERDAD

// `window.CPII` → namespace legado con session e i18n, en transición hacia AIP/Skeleton.  
// `window.LuxI18n` → diccionario global de traducciones, fuente de verdad para R4.  
// `window.eventBus` → bus legado obsoleto, persistente solo en partes no migradas.  
// `globalBus` → bus de eventos usado en el resto del sistema, en camino de deprecación.  
// `document.dispatchEvent` → mecanismo canónico de comunicación entre órbitas.  
// `window.Skeleton` → namespace canónico actual con PassportEngine, UniversalGatekeeper, Registry, i18n y session.  
// `window.AIP` → namespace intermedio con session, `i18n.t()` y `bus.emit()`.  
// `offeringConfig` → objeto de configuración inyectado que define el offering actual.  
// `IntegrityScore` → entero 0-100 calculado por IntegrityEngine; umbral bancable 60.  
// `window.Store` → singleton de StoreService; estado central, actualmente inoperativo como `.bak`.  
// `window.Gatekeeper` → orquestador base de acceso determinista.  
// `PassportEngine` → motor de pasaportes, autentica y persiste sesión.  
// `TripleEsclusa` → máquina de estado de compliance en etapa stub.  
// `SceneManager` → orquestador de visibilidad de órbitas y gates.  
// `ThemeEngine` → inyector de variables CSS en la raíz del documento.  
// `root` → espacio de tokens CSS, única fuente permitida de colores.  
// `sessionStorage` → almacén de `skeletonpassport` y persistencia de sesión.  
// `localStorage` → no usado oficialmente; se considera fuga potencial.  
// `data-i18n` → ancla HTML global para internacionalización R4.  
// `BR-XX.md` → conjunto de contratos de referencia bajo custodia.  
// `tree-AIP.txt` → coordenada física validada, snapshot manual frecuentemente desactualizado.
